# コードベース概要と改善タスク提案

## コードベース概要

- **フロントエンド構成**: 単一ページアプリで、`index.html` + `css/style.css` + `js/main.js` で構成。
- **主機能**:
  - ファイルサイズ/回線速度からダウンロード時間を計算（`js/main.js`）。
  - 日英切り替え（`data/ja.json`, `data/en.json`）。
  - 設定（テーマ・アクセントカラー・単位基準）を Cookie に保存。
  - 計算履歴（最大5件）を Cookie に保存。
  - PWA 対応（`manifest.json`, `js/service-worker.js`）。
- **補助ページ**: `pages/privacy.html` + `data/privacy_*.json` でプライバシーポリシーを表示。
- **テスト現状**: 自動テストファイルが存在しない。

---

## 提案タスク（4件）

### 1) 入力ミス修正タスク
**内容**: 英語ラベル `Gray Dark` を `Dark Gray`（または `Gray`）に統一修正する。  
**対象候補**: `index.html`, `pages/privacy.html`。  
**根拠**: UI 文言として不自然で、単純なテキスト入力ミスと判断できる。

### 2) バグ修正タスク
**内容**: 速度単位変換（Kbps/Gbps→Mbps）で `1024` 固定になっている点を、サイズ単位と同じ設定値（`sizeUnitBase`: 1000/1024）に連動させるか、UI 仕様として明示的に SI(1000) 固定にする。

- 現状: サイズ換算は `sizeUnitBase` を使うが、速度換算は `1024` 固定。
- 影響: `1Gbps` 計算時に SI 前提のユーザー期待値と差が出る可能性。

**対象候補**: `js/main.js` の `calculateDownloadTime()`。

### 3) コメント/ドキュメント矛盾修正タスク
**内容**: プライバシーポリシー内の「履歴と設定はローカルストレージ保存」という説明を、実装の Cookie 保存に合わせて修正する（もしくは実装を localStorage へ移行して説明に合わせる）。

- 現状実装: `setCookie/getCookie` を履歴・設定保存に利用。
- 現状文書: `data/privacy_ja.json`, `data/privacy_en.json` で local storage と記載。

### 4) テスト改善タスク
**内容**: 変換ロジックを純関数化して、最低限の自動テストを追加する。

- 追加対象（例）:
  - ファイルサイズ単位変換（MB/GB/TB, 1000/1024）
  - 速度単位変換（Kbps/Mbps/Gbps）
  - 計算時間算出（境界値: 0, 小数, 大容量）
  - 表示整形（`displayResult` 相当）
- 期待効果:
  - 単位系の退行バグを防止。
  - 今回のバグ修正（タスク2）を再発防止可能。
